# MindVerse XR

## Problem

Flat 2D mind maps lose the multi-dimensional spatial relationships people
naturally form when thinking. MindVerse XR moves mind mapping into 3D MR space
on HoloLens 2 so users can grab, arrange, and group nodes with their hands.

## What I built

I joined the project in **April 2026** as a refactorer and contributor at the
Metaverse ITB lab. Work focused on three areas:

- **Interaction module consolidation** — unified scattered grabable-object
  scripts behind a single, predictable interface so adding new node types no
  longer requires touching five files.
- **Command-pattern undo/redo** — every state-changing interaction emits a
  Command object so users can reverse moves naturally. Critical for MR, where
  accidental hand input is common.
- **Object pooling for grabables** — eliminates per-frame GC spikes that cause
  HoloLens motion sickness during heavy sessions.

I also demoed the application to researchers and visitors at 5+ lab events.

> Note: my refactor work currently lives on local feature branches
> (`refactor/system-wide`, `fix/*`) pending review before merging into the
> shared `main`.

## Outcomes

- 5+ live demo sessions on HoloLens 2 with positive feedback from researchers
  and guests.
- Cleaner module boundaries that improved onboarding time for newer
  contributors.

## Tech highlights

- **Command pattern** for irreversible-feeling MR interactions — users can
  always step back.
- **Object pooling** to keep GC out of the per-frame hot path.
- **Synchronized multi-feedback** via More Mountains Feel + Pixel Crusher so
  visual, audio, and haptic cues stay coherent across grabs, snaps, and
  releases.
